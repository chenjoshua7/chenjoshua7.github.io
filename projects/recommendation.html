MovieLens Recommendation System

Special thanks to my wonderful project partners: Jonny Codd, Carlos Perez, and Rafael Gallegos.

Note: As discussed later, our results for recommendations illuminated some interesting information about the Cold Start problem. I am currently working on an item-to-item extension on this problem. Come back soon to see how I tackle this problem.

Our final project can be found here.

To get a stronger understanding of Recommendation Systems methodologies, we studied the application of different recommendation systems on the MovieLens DataSet. We looked at the effectiveness of implementing ProbS, HeatS, and Collaborative Filtering as well as the network affects these recommendations had over iterations.

The results are listed below. We chose Precision - \(\frac{TP + FP}{TP}\) - as our main evaluation metric. When recommending movies, we considered it to be much more important to recommend only movies a person would like, rather than recommending all movies a person would like. However, this would not be case for, say, an e-commerce business where a FP results is relatively harmless but a FN is lost income.

Discussion

Collaborative Filtering returned a very high precision but we had a problem.

Every single movie reocmmended to User 314 was a highly rated movie from before 2000, despite a large majority of the movies in the dataset being post 2000. This is a perfect representation of the Cold Start problem.

Here we see this problem represented for through running iterations of the Prob S recommendations. Already popular movies will get disproportionately recommended over time, with smaller movies never getting recommended. Heat S, as seen below, sacrifices precision. However, it allows for movies to be more equitably recommended.
